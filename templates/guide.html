First login

The appliance ships with a single administrator account. On first sign-in you are forced to change the password before any other page is reachable. Use a strong, unique password — sessions are bcrypt-backed and expire after 8 hours (30 min idle).

For production, enable two-factor authentication (Profile → Two-Factor) and upload a real TLS certificate (below). The default certificate is self-signed.

Data-plane modes

The forwarding engine runs as a separate, root data-plane process. The backend is fixed at install/boot time via the BROKER_MODE setting and shown in the sidebar pill:

  • libpcap — broadest NIC compatibility; one capture thread per input interface.
  • afxdp — zero-copy AF_XDP fast path on driver-mode XDP NICs (i40e/igc/ice/mlx5); one worker per RX queue. Interfaces stay kernel-visible, so the UI sees them by name. Recommended default.
  • dpdk — DPDK PMD; one worker lcore per port. Requires operator hugepage + NIC binding (vfio-pci), and bound NICs leave the kernel — see the deployment notes. The management NIC must never be DPDK-bound.

Start/stop the data plane from the dashboard. Rules can be added/edited/removed at runtime without restarting — the engine binds all data-plane ports up front.

Rules & topology

A rule forwards traffic from one input port to one output port, optionally filtered and manipulated. Fields: input/output interface, protocol, source/dest port, TCP flags, VLAN match, source/dest IP (CIDR), source/dest MAC, payload string match, BPF filter, and an exclude (invert) flag. Manipulation: VLAN add/remove/change and truncation.

The management interface is auto-detected and removed from every port selector, so you cannot accidentally forward through the port you administer the box on. The topology diagram visualises active links.

Mirror / Load-balance / SSL / Dedup / DPI / Masking

  • Mirror — copy one source to several destinations (SPAN/TAP aggregation).
  • Load-balance — spread one input across several outputs.
  • SSL decrypt chain — route encrypted → inspection tool → re-inject, with an optional BPF pre-filter.
  • Deduplication — drop near-duplicate frames within a time window (CRC over the first N bytes).
  • DPI — classify L7/OT protocols (HTTP, TLS, DNS, SSH, Modbus, DNP3, S7comm, …) and route by protocol.
  • Masking — anonymise IP/MAC/payload or regex-replace PII before forwarding.

Monitoring & alerts

The dashboard shows live CPU/memory/uptime and per-port throughput. Configure threshold alerts (drop rate, link down, CPU, memory) and deliver them by email (SMTP or Microsoft Graph), syslog, or SNMP (the agent exposes status to Zabbix/Nagios/PRTG/LibreNMS). Outbound webhooks/SMTP are constrained to non-internal targets to prevent SSRF.

Cluster

Run several brokers as controller + nodes. Nodes send authenticated heartbeats (256-bit shared secret, constant-time check) to the controller, which aggregates status. Copy the controller secret into each node's cluster config.

Captures

Start on-demand packet captures (tcpdump) per interface with an optional filter; download the resulting PCAP. Capture files have unguessable IDs and are admin-only.

Backups & firmware

Backups export/import rules and config as a signed bundle. Firmware upgrades the data-plane binary — uploads MUST be vendor-signed (Ed25519); rollback re-verifies the stored signature before reinstalling. Unsigned binaries are refused.

TLS certificate, FQDN & branding

Under Settings you can: set the product name and upload a logo (PNG/JPEG/GIF/ICO; SVG is rejected); set the FQDN; and manage TLS — upload your own PEM certificate + key, or regenerate a self-signed certificate that includes the FQDN. TLS changes apply on the next connection without a restart. The server runs TLS 1.3 only.

Users & 2FA

Admins manage users (admin/user roles). Admin-created accounts must change their password on first login. Each user can enable TOTP two-factor (Google Authenticator/Authy); codes are single-use (replay-protected).

License

The product is hardware-ID locked (Ed25519 vendor-signed). Upload your license under Admin → License; the dashboard shows type, customer and expiry.

Troubleshooting

  • Data plane won't start — check the broker log (Logs page / packet_broker.log). For afxdp/dpdk confirm NIC capability; for dpdk confirm hugepages + NIC binding.
  • Locked out after attaching a port — never select the management interface in a rule (it is auto-excluded). For dpdk, never vfio-bind the management NIC.
  • Browser certificate warning — expected with the default self-signed cert; upload a CA-signed cert or set the FQDN and regenerate.
  • Forgot password — reset requires console/physical access to the appliance.
